## Conversation Flow & Decisions

### 1. Initial Concept Discussion

**User's Vision:**
- Website for kids to generate stories
- Kids can only enter their names (security feature)
- Everything else is dropdown selections:
  - Pronouns
  - Age (3-9)
  - Story world
  - Antagonist/challenge
  - Story length (short, medium, long with max 4-5 paragraphs)
  - Reading level based on age

**Security Rationale:**
Limiting input to dropdowns prevents prompt engineering hacks and ensures kid-safe content.

### 5. Security & Safety Features Discussion

**Critical Decision:** No free-text prompts except name input

**Rationale:**
- Prevents prompt injection attacks
- Ensures content stays appropriate
- Protects against malicious input
- Simplifies moderation

**Implementation:**
- All selections via dropdowns/visual buttons
- Only the name field accepts text input
- Age, pronouns, world, challenge, length all pre-defined options

### 11. Content Guidelines for AI Story Generation

**Safety Rules:**
1. Always positive and empowering
2. Age-appropriate vocabulary matched to selected age
3. No scary content - challenges are fun, not frightening
4. Respect chosen pronouns
5. Educational value (friendship, problem-solving, courage)

**Story Structure Template:**
```
Opening: Introduce [childName] in [storyWorld]
Challenge: [childName] meets [challenge]
Adventure: [childName] uses [positive trait] to overcome obstacle
Resolution: Success through [bravery/kindness/cleverness]
Closing: Empowering message about [childName]'s abilities
```

**Vocabulary by Age:**
- Ages 3-4: Simple words, short sentences, repetition
- Ages 5-6: Basic descriptive words, slightly longer sentences
- Ages 7-9: More complex vocabulary, varied sentence structure

**Length Guidelines:**
- Short: 50-100 words (bedtime for younger kids)
- Medium: 150-250 words (engaging but not too long)
- Long: 300-400 words (immersive adventure)
